## 🔴 严重问题

### 1. 死循环问题

**问题描述**:
- 迭代 3-6 一直在循环，但没有待处理的视频
- 待下载视频数为 2，但下载阶段显示"没有待下载的视频"
- 编排脚本没有正确处理这种情况，导致无限循环

**根本原因**:
- videos.json 中有 2 个视频状态为 "pending"
- 但这两个视频的 author_id 与博主列表中的 author_id 不匹配
- stage4 正确地跳过了这些视频，但编排脚本没有识别这种情况

**影响**:
- 浪费系统资源
- 无法自动终止
- 每次迭代耗时约 20 秒（下载 17s + 转录 3s）

**建议修复**:
```javascript
// 在 pipeline-orchestrator.js 中添加检测逻辑
if (downloadable.length === 0 && stats.pending > 0) {
  log('⚠️ 检测到有待下载视频但无法下载，可能存在数据不一致');
  log('   建议运行数据一致性检查: opencli douyin-videos consistency-check');
  break;
}
```

### 2. author_id 不匹配问题

**问题描述**:
- 两个视频的 author_id 与博主列表中的 author_id 不一致
- 导致 stage4 无法识别这些视频的作者，从而跳过下载

**问题视频**:

| aweme_id | author | video author_id | blogger author_id | 差异 |
|----------|--------|-----------------|-------------------|------|
| 7624450000738602249 | 创业老王不乱讲 | 2487558162360585 | 2487558162260585 | 中间数字不同 |
| 7630395483273493888 | 薛辉小清新 | 67904681231 | 67904681221 | 末尾数字不同 |

**根本原因**:
- Stage2 在收集视频时，使用了抖音 API 返回的 author_id
- 但博主列表中的 author_id 可能来自不同的数据源
- 导致 author_id 不一致

**建议修复**:
1. **短期**: 将这两个视频标记为 "inaccessible"，避免死循环
2. **长期**: 修复 author_id 匹配逻辑，使用 sec_uid 或博主名称作为主要匹配字段
