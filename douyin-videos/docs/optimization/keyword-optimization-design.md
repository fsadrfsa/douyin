## 一、功能概述

### 1.1 目的

寻找符合主题的关键词，优化关键词列表，提高搜索效果。

### 1.2 主题定义

线上项目、创业、副业、赚钱、商业信息、AI落地项目

### 1.3 核心流程

```
关键词搜索 → 发现博主 → 采集视频 → 提取口播文案 → 提取项目信息 → 评估项目质量
```

### 1.4 触发条件

智能体在以下情况下执行：

| 触发场景 | 执行动作 | 说明 |
|----------|----------|------|
| **阶段三完成后** | 自动执行优化 | 阶段三（内容分析）完成后，数据完整，可进行准确评估 |
| **手动触发** | 用户执行命令时运行 | 用户主动执行优化命令 |
| **定期执行** | 每天定时运行 | 分析前一天的数据，生成优化建议 |

## 三、评估逻辑

### 3.1 评估用户关键词效果

#### 评估指标

| 指标 | 数据源 | 计算 |
|------|--------|------|
| **找到博主数** | bloggers.json | 该关键词对应的博主数量 |
| **符合主题博主数** | results/ | 领域匹配主题的博主数量 |
| **符合主题比例** | 计算 | 符合主题博主数 / 找到博主数 |
| **内容关键词匹配度** | statistics.json（待实现） | 该关键词相关的内容关键词与主题的匹配度 |

#### 判断逻辑

**主题关键词**（可根据实际情况扩展）：
- 创业、副业、赚钱、项目、商业、AI落地、变现、创业项目、副业项目、赚钱项目

**判断博主是否符合主题**：

```
1. 读取博主的分析结果
2. 检查"领域"字段是否包含主题关键词
3. 检查"关键词"字段是否包含主题关键词
4. 检查"博主一句话总结"是否包含主题关键词
5. 如果任一匹配，则判定为符合主题
```

**边界情况处理**：

```
如果找到博主数为0：
  - 符合主题比例记为0%
  - 状态标记为"无数据"
```

**内容关键词匹配度计算**（待 statistics.json 实现后启用）：

```
1. 从 statistics.json 中读取该关键词相关的内容关键词
2. 计算内容关键词中包含主题关键词的比例
3. 匹配度 = 包含主题关键词的内容关键词数 / 总内容关键词数
```

#### 效果评分

| 符合主题比例 | 评分 | 状态 |
|--------------|------|------|
| ≥70% | 优秀 | ✅ 保持观察 |
| 50-69% | 良好 | ⚠️ 可优化 |
| 30-49% | 一般 | ⚠️ 需优化 |
| <30% | 较差 | ❌ 建议替换 |
| 无数据 | 无数据 | ⏸️ 待评估 |

---

### 3.2 评估抖音推荐关键词价值

#### 评估指标

| 指标 | 数据源 | 计算 |
|------|--------|------|
| **关键词相关性** | discovered-keywords.json | 关键词本身是否包含主题关键词 |
| **来源关键词效果** | search-keywords.json | source_keyword 的效果评分 |

#### 判断逻辑

```
1. 检查关键词本身是否包含主题关键词
2. 检查来源关键词（source_keyword）的效果评分
3. 如果关键词相关且来源关键词效果好，则推荐添加
```

#### 推荐优先级

| 条件 | 优先级 |
|------|--------|
| 关键词相关 + 来源关键词效果优秀 | 高 |
| 关键词相关 + 来源关键词效果良好 | 中 |
| 关键词相关 + 来源关键词效果一般 | 低 |
| 关键词不相关 | 不推荐 |

## 四、验收标准

### 4.1 关键词优化验收标准

#### 核心指标

| 指标 | 优化前 | 优化后 | 验收标准 |
|------|--------|--------|----------|
| **平均符合主题比例** | 当前值 | 优化后值 | 提升 ≥10% |
| **优质关键词占比** | 当前值 | 优化后值 | 提升 ≥15% |
| **新增符合主题博主数** | - | 新增值 | ≥5个 |

**优质关键词定义**：评分≥良好的关键词（符合主题比例≥50%）

**优质关键词占比计算**：
```
优质关键词占比 = 优质关键词数 / 总关键词数
```

#### 验收流程

```
步骤1：记录优化前的数据
  - 记录每个关键词的符合主题比例
  - 记录优质关键词占比

步骤2：执行优化
  - 添加推荐关键词
  - 标记效果差的关键词

步骤3：等待新一轮搜索和分析
  - 用户手动执行阶段一、二、三
  - 等待执行完成后，收集新的数据
  - 注意：优化后需要重新执行流程才能验证效果

步骤4：对比优化前后数据
  - 计算平均符合主题比例提升
  - 计算优质关键词占比提升
  - 统计新增符合主题博主数

步骤5：判断是否通过验收
  - 如果所有指标达标，则通过
  - 如果有指标未达标，则继续优化
```

## 五、边界条件

### 5.1 不执行优化的情况

| 边界条件 | 说明 | 处理方式 |
|----------|------|----------|
| **数据不足** | 分析结果 <10个 | 不执行优化，记录原因 |
| **所有关键词效果优秀** | 平均符合主题比例 ≥80% | 不执行优化，记录"已达最优" |
| **阶段三未完成** | pipeline-state.json 显示阶段三未完成 | 不执行优化，提示用户先完成阶段三 |
| **关键词数量已达上限** | 见第六章约束条件 | 见第六章约束条件 |

### 5.2 优化无效的情况

| 边界条件 | 说明 | 处理方式 |
|----------|------|----------|
| **连续3次优化无效果** | 验收标准连续3次未达标 | 停止自动优化，记录问题，建议人工介入 |
| **抖音推荐关键词已全部添加** | discovered-keywords.json 中所有关键词已添加 | 不添加新关键词，建议用户手动添加 |
| **主题关键词不明确** | 主题定义模糊或冲突 | 不执行优化，提示用户明确主题 |

### 5.3 需要人工介入的情况

| 边界条件 | 说明 | 处理方式 |
|----------|------|----------|
| **关键词效果持续下降** | 优化后效果反而下降 | 记录问题，建议人工检查 |
| **博主质量判断异常** | 符合主题比例异常高（>95%）或低（<5%） | 记录问题，建议人工检查判断逻辑 |
| **数据异常** | 文件损坏、数据丢失、格式错误 | 记录问题，不执行优化 |

### 5.4 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| **文件读取失败** | 记录错误，跳过该文件 |
| **数据格式错误** | 记录错误，使用默认值 |
| **写入失败** | 记录错误，不更新文件，保留原数据 |

## 六、约束条件

### 6.1 数量约束

| 约束类型 | 数量 | 说明 |
|----------|------|------|
| **最小数量** | ≥5个 | 保证有足够的关键词进行搜索 |
| **最大数量** | ≤8个 | 避免搜索时间过长 |
| **推荐数量** | 5-8个 | 平衡搜索效率和覆盖率 |

### 6.2 长度约束

| 约束类型 | 长度 | 说明 |
|----------|------|------|
| **最小长度** | ≥2个字符 | 避免单字关键词 |
| **最大长度** | ≤20个字符 | 避免过长的关键词 |

### 6.3 格式约束

| 约束类型 | 规则 | 说明 |
|----------|------|------|
| **允许的字符** | 中文、英文、数字、空格 | 避免特殊字符 |
| **禁止的字符** | 特殊符号、表情符号 | 保证搜索兼容性 |

### 6.4 唯一性约束

| 约束类型 | 规则 | 说明 |
|----------|------|------|
| **关键词唯一** | 不能重复 | 避免重复搜索 |
| **不区分大小写** | "AI工具" 和 "ai工具" 视为相同 | 避免重复 |

### 6.5 边界处理

| 情况 | 处理方式 |
|------|----------|
| **关键词数量 <5个** | 不执行优化，提示用户添加更多关键词 |
| **关键词数量 >8个** | 不添加新关键词，建议删除效果差的关键词 |
| **关键词重复** | 自动去重 |
| **关键词格式错误** | 记录问题，跳过该关键词 |

## 七、工作流程

### 7.1 智能体工作流程

```
步骤1：读取数据
  输入：
    - search-keywords.json（用户关键词）
    - discovered-keywords.json（抖音推荐关键词）
    - bloggers.json（博主列表）
    - results/（分析结果）
    - statistics.json（统计信息，待实现）
  输出：
    - 数据加载成功/失败状态

步骤2：评估用户关键词效果
  输入：
    - 用户关键词列表
    - 博主列表
    - 分析结果
  输出：
    - 每个关键词的效果评分
    - 符合主题博主列表

步骤3：评估抖音推荐关键词价值
  输入：
    - discovered-keywords.json
    - 用户关键词效果评分
  输出：
    - 推荐添加的关键词列表（按优先级排序）

步骤4：生成优化建议
  输入：
    - 用户关键词效果评估结果
    - 推荐添加的关键词列表
  输出：
    - 优化建议报告

步骤5：执行优化
  输入：
    - 优化建议
  输出：
    - 更新后的 search-keywords.json
    - 优化报告文件
```
